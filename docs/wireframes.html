01 Where the app fits

The Volume Planner is one step in a five-step pipeline. Mehrdad's team runs the pipeline weekly. The app handles the human review and adjustment step only — it does not replace the R model or the data pulls.

01
Courier Ops
Daily volume pulled from QlikView each Monday
02
FMR Daily DB
Pre-processed, cleaned, appended to history
03
R Forecast Model
STL + ARIMA fits seasonal trend. Outputs forecast CSV
04
Volume Planner
Planners review, adjust, validate daily shapes, export
05
Final Forecast
Finalized CSV feeds daily distribution model
What the app does not do: run the R model, pull from Courier Ops directly, store data in a central database, or handle multi-user authentication. These are all separate concerns.

How the table selection works

Click any cell to select it. The blue highlight and cell reference in the toolbar update immediately. Shift+click extends selection. Ctrl+click adds individual cells. When you have weeks selected, the blue selection KPI bar appears above the table showing YoY stats for only those weeks. Ctrl+Shift+Y refreshes it.

How operations work

Choose an operation (% Increase, Set to, Add), type a number, press Enter or click Apply. If you set a week to 500K and then add 500K, the result is 1,000K — not the original plus 500K. The original forecast line stays on the chart as a grey dashed line whenever you have unsaved modifications. Revert Selected steps back one operation at a time. Revert All resets everything.

Prophet 2027 overlay

The Prophet 2027 line uses the same x-axis positions as the 2026 forecast — week 27 of 2027 sits directly above week 27 of 2026. This means you can visually compare the two years' peak timing and amplitude without any axis shifting.

The overlay system — how it works

Type a year and week in the Overlay inputs, click Add. That week's distribution fetches from the uploaded file and draws as a dashed line on the same chart. Add multiple overlays — each gets a different color with a labeled tag you can remove. The orange bold line is the normalized aggregated average across all overlays. This is how Mehrdad derives event distributions: pull 2-3 historical weeks with the same holiday weekday pattern, see them overlaid, validate the orange average looks right, save it.

Auto-loading for event weeks

When you navigate to a known event week (Canada Day W27, Labour Day W36, Thanksgiving W41, Remembrance Day W45, Peak W47-49, Christmas W51-52), the app looks up the saved terminal preferences or global event map and loads the base weeks as overlays automatically. You do not need to type them — the comparison appears immediately.

The edit table

Change any day's percentage directly. The Total row turns green when values sum to exactly 100%, red otherwise. Normalize to 100% proportionally adjusts all days. Save & Download CSV writes the change to memory, records it in the distribution change log (with before/after values and timestamp), and downloads the updated CSV. That CSV is re-uploaded next session.

The output-becomes-input pattern: every CSV the app exports is what gets re-uploaded at the start of the next session. The distribution file carries its own change history as comment rows at the bottom. There is no database. State lives in files.

06 What works today vs what is next

Built and working

  • Weekly forecast review chart with all line types
  • Prophet 2027 overlay on same x-axis as 2026
  • Sequential operations (set 500, add 500 = 1000)
  • Stack-based revert, one step at a time
  • Change log sidebar with timestamps
  • Selection-range KPI bar (Ctrl+Shift+Y)
  • Daily page: Sun-Sat line chart with % / Volume toggle
  • Overlay system: multiple base weeks, same canvas, orange aggregate
  • Auto-load base weeks for event weeks
  • Edit table with normalize to 100%
  • Distribution change log (before/after, timestamp)
  • Event preferences save / export / import CSV
  • Export finalized weekly forecast CSV

Not yet built

  • Cell-level selection (currently row-level)
  • Chart week-range highlight on table selection
  • Chart PNG export with metadata in image
  • Load previous vs start fresh modal (daily page)
  • AWS deployment with S3 file persistence
  • User login / per-terminal access control
  • SharePoint integration for final export
  • Running the R model from within the app
07 Future features and AI integration

Features roughly ordered by value to Mehrdad's team. The first three are already in the codebase, just not fully exposed in UI.

High priority

AWS deployment + S3

Remove the need to re-upload files each session. Distribution file loads automatically on startup. ~$20/month on EC2 t3.small.

High priority

Cell-level selection

Mehrdad said: "it should be for a cell, not the full row." Shift+click to extend, Ctrl+click to add individual cells across any column.

High priority

Chart week highlight

When weeks are selected in the table, shade those columns on the chart. Makes the table-chart link obvious for field engineers.

Medium

Version history

Timestamp-versioned CSV exports. "Load previous" modal at daily page startup. Each file carries its full edit history already — just needs a UI for browsing versions.

Medium

Terminal access control

Field engineers log in and see only their terminals. Edits visible to everyone but scoped to their division. Requires Purolator SSO integration.

Medium

Ensemble forecast

Weighted average of Prophet + the R model output. Slider in sidebar to adjust weights. Usually more accurate than either model alone.

Future

Gemini anomaly explanation

When a week's YoY is an outlier, call Gemini API with context. Returns a plain-English explanation. "W27 is Canada Day — Monday volume typically drops 80%."

Future

Gemini event similarity

Given a new holiday weekday pattern, Gemini identifies the best 2-3 historical base weeks to use for the daily distribution — replacing the manual lookup process.

Future

TradingView-style chart

Zoom, pan, crosshair hover showing all lines at a week. Right now the chart is static height. A zoomable canvas would help during peak week analysis.

08 Maintenance and handover

Annual tasks (January)

  • Update the historical daily distribution parquet file — add the completed prior year
  • Update mock_event_map.csv with the new year's holiday weeks and which weekday each falls on
  • Identify 2-3 historical base weeks for each event (same holiday weekday pattern)
  • Verify Prophet model config in prophet_pipeline/config.py is still appropriate

Weekly tasks (Monday)

  • Mehrdad runs the Courier Ops pull and R model externally
  • Upload Actual.xlsx and Forecast.xlsx to the app
  • Review chart, make adjustments, validate daily distributions for event weeks
  • Export finalized CSV and submit to planning
